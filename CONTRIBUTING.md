# Contributing to Smart Dev Tools

We love pull requests from the community! 

This repository aims to be the highest quality collection of cheatsheets and guides for developers. By contributing, you're helping thousands of developers save time and avoid bugs.

## How to Contribute

1. **Fix typos or technical errors**: If you spot an inaccuracy in any guide, please open a PR.
2. **Add examples**: Helpful code snippets or common use cases are always welcome.
3. **Propose new cheatsheets**: Have a great idea for a new developer tool guide? Open an issue first to discuss.

## Guidelines

- Keep the content **accurate and factually verifiable**.
- Use clear, concise, and professional language.
- Ensure all code blocks are properly formatted and tested.
- Do not add spammy or self-promotional links.
- Maintain the current formatting style (Markdown headers, tables, etc.).

### Setting up locally
You don't need any complex local setup. Just fork the repository, make your changes to the markdown files in your favorite text editor, and open a Pull Request.

Thank you for contributing!
