## What is a Checksum?
A checksum (or hash) is a fixed-size string of letters and numbers generated by a mathematical algorithm from a file or string. Even changing a single pixel in an image or one space in a text file will completely change the resulting checksum.

They are used to verify **data integrity** (proving a file hasn't been corrupted during download) and for **security** (storing passwords securely without saving the plaintext).

## Hash Algorithm Comparison

| Algorithm | Bit Length | Speed | Collision Risk | Security Status | Primary Use Case |
|-----------|------------|-------|----------------|-----------------|------------------|
| **MD5** | 128-bit | Very Fast | High | ❌ Deprecated for Security | Fast file deduplication, basic integrity checks |
| **SHA-1** | 160-bit | Fast | Medium | ❌ Deprecated for Security | Git commits (legacy), file integrity |
| **SHA-256** | 256-bit | Slower | Extremely Low | ✅ Highly Secure | Passwords, Blockchain, SSL Certificates, Signatures |
| **SHA-512** | 512-bit | Slower | Extremely Low | ✅ Highly Secure | High-security enterprise systems |

## Common Mistakes / Gotchas
1. **Using MD5 for Passwords:** MD5 is easily broken using "rainbow tables" (pre-calculated dictionaries of hashes). It should *never* be used for hashing passwords or securing sensitive data.
2. **Confusing Encryption with Hashing:** Encryption is a two-way function (you can decrypt the data if you have the key). Hashing is a one-way function. You cannot "reverse" a SHA-256 hash to get the original file back.
3. **Salts:** When hashing passwords, always add a random "salt" string to the password before hashing it to prevent dictionary attacks.

## When to Use This Tool
* Verifying that an ISO or large software package downloaded perfectly from the internet by comparing its hash against the vendor's provided MD5/SHA256 string.
* Generating a unique cache-busting key for a file upload in S3 based on the file's contents.
* Quickly checking if two files are exactly identical without comparing them byte-by-byte.
